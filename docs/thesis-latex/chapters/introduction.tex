\chapter{Bevezetes}
\label{ch:introduction}

Ez a szakdolgozat egy receptkezelo webalkalmazas tervezeset es megvalositasat mutatja be. A rendszer celja, hogy a felhasznalok egy korszeru webes feluleten tudjak rogzitni, keresni, ertekelni es megosztani receptjeiket, mikozben a megoldas kulon figyelmet fordit az allergeninformaciok kezelesere es a jol strukturalt API-alapu integraciora. Az alkalmazas technikai alapjat egy Vue alapokra epulo kliens, egy ASP.NET Core Web API szolgaltatas, valamint egy relacios adatbazis adja \cite{vue,aspnetcore,postgresql}.

\section{Motivacio}

A receptek digitális kezelese jellemzoen tobb kulonallo igenyt egyesit. Fontos a gyors kereses, a reszletes receptleiras, a felhasznaloi visszajelzesek tamogatasa, valamint az, hogy az etelallergiaval elo felhasznalok is atlathato informaciot kapjanak az osszetevokrol. Az alkalmazas egyetlen rendszerben kivanja osszefogni ezeket a funkcionalitasokat, tovabba alkalmas alapot szeretne adni a kesobbi bovitesekhez is.

\section{A dolgozat celja}

A dolgozat ket fo celra epul:

\begin{itemize}
	\item bemutatni a rendszerrel szemben megfogalmazott funkcionális es nem funkcionális kovetelmenyeket,
	\item dokumentalni az alkalmazott architekturalis es implementacios donteseket ugy, hogy azok visszavezethatok legyenek a projektben megjeleno tenyleges megoldasokra.
\end{itemize}

Ennek megfeleloen a dolgozat nem csupan egy altalanos termekbemutato, hanem egy fejlesztesi dokumentacio is, amely lefedi a kovetelmenyeket, a rendszertervet, az implementacio fo reszeit, valamint a validacio es teszteles modszereit.
