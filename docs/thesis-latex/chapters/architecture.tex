Az alkalmazás kettő fő részből áll: egy ASP.NET Core alapú backend szolgáltatásból, egy Vue és TypeScript alapú frontend egyoldalas alkalmazásból. Fontos hogy ezen részek önmagukban is megállják a helyüket, de együtt egy teljes funkcionalitást nyújtó rendszert alkotnak. Ennek megvalósásában az stabil API-kapcsolat, a jól definiált adatmodell, valamint a felhasználói élményre fókuszáló frontend kialakítása a fő célok közé tartozik. 

\subsection{Funkcionális követelmények}\label{ch:requirements}

Funcionális követelmények tekintetében a projekt fő területei a következők:

\begin{itemize}
	\item Felhasználó- és profilkezelés: regisztráció, bejelentkezés, profil megjelenítése és módosítása.
	\item Receptkezelés: új recept létrehozása, szerkesztése, törlése, részletes receptnézet.
	\item Keresés és szűrés: receptlista böngészése, kulcsszavas keresés, szűrés allergének alapján.
	\item Közösségi funkciók: receptek értékelése, kommentek rögzítése és törlése.
\end{itemize}

\subsection{Nem funkcionális követelmények}

A rendszerrel szemben a funkciókon túl a következő nem funkcionális elvárások is megjelennek:

\begin{itemize}
	\item átlátható és reszponzív felhasználói felület,
	\item jól dokumentált, stabilan használható HTTP API,
	\item olyan adatmodell, amely a recept-, komment- és felhasználói adatok közötti kapcsolatokat konzisztensen kezeli,
	\item továbbfejleszthető komponensfelosztás a frontend és a backend oldalon,
	\item fejlesztői munkát segítő konténeres indítási és build környezet \cite{docker}.
\end{itemize}

\subsection{Architekturális áttekintés}

A projekt réteges, webalkalmazás-jellegű architektúrára épül. A felhasználó egy böngészőben futó kliensalkalmazással lép kapcsolatba, amely HTTP hívásokon keresztül éri el a backend szolgáltatást. A backend végzi az üzleti logika, az adatelérés, a hitelesítéshez kapcsolódó folyamatok, valamint az API-dokumentáció kiszolgálásának fő részét. Az adatok perzisztálását egy relációs adatbázis biztosítja \cite{vue,aspnetcore,postgresql}.

A fő komponensek:

\begin{itemize}
	\item Frontend: Vue alapú egyoldalas alkalmazás, amely a felhasználói folyamatokat és az oldalak közti navigációt kezeli.
	\item API kliens: OpenAPI alapján generálható klienscsomag, amely a frontend és a backend közötti szerződést teszi követhetővé.
	\item Backend: ASP.NET Core Web API, amely kontroller-, szolgáltatás- és repository-szintű felelősségekre tagolódik.
	\item Adatbázis: PostgreSQL kompatibilis adatbázis, amelyet az alkalmazás entitásaihoz és kapcsolataihoz igazított séma szolgál ki.
	\item Infrastruktúra: Docker Compose alapú helyi futtatási környezet, amely támogatja a fejlesztést és a rendszer összeépítését \cite{docker}.
\end{itemize}

A repository szerkezete is tükrözi a komponensfelosztást. A \texttt{src/backend} könyvtár a .NET oldali kódot, a \texttt{src/frontend} a kliensalkalmazást és a generálható API klienst, a \texttt{docs} pedig a projekt dokumentációs anyagait tartalmazza.
