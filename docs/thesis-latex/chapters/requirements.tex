\section{Nem funkcionalis kovetelmenyek}

A rendszerrel szemben a funkciokon tul a kovetkezo nem funkcionalis elvarasok is megjelennek:

\begin{itemize}
	\item atlathato es reszponziv felhasznaloi felulet,
	\item jol dokumentalt, stabilan hasznalhato HTTP API,
	\item olyan adatmodell, amely a recept-, komment- es felhasznaloi adatok kozotti kapcsolatokat konzisztensen kezeli,
	\item tovabbfejlesztheto komponensfelosztas a frontend es a backend oldalon,
	\item fejlesztoi munkat segito konteneres inditasi es build kornyezet \cite{docker}.
\end{itemize}

\section{Dokumentacios kovetelmenyek}

Mivel a projekt szakdolgozati keretben keszul, maga a dokumentacio is kulon kovetelmenynek tekintheto. A dolgozatnak le kell vezetnie, hogy a rendszer milyen felhasznaloi igenyekre ad valaszt, hogyan illeszkednek egymashoz a fo komponensek, es mely implementacios dontesek tamasztjak ala a megfogalmazott celokat. Kiemelten fontos tovabba, hogy az OpenAPI specifikacio es a valos rendszerkozotti kapcsolat egyertelmuen kovetheto legyen \cite{openapi}.
